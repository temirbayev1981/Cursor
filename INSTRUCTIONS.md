## 1. Обзор системы

**HandymanOS AI** — корпоративная SaaS-платформа для компаний, которые выполняют ремонт, техническое обслуживание объектов недвижимости и смежные услуги. Система объединяет CRM, операционное управление заказами, финансы, склад, автопарк, диспетчеризацию и ИИ-автоматизацию в одном интерфейсе.

### Ключевые возможности

| Направление | Что даёт система |
|-------------|------------------|
| Операции | Заказы, наряды, расписание, диспетчерская Kanban, маршруты |
| Продажи | Сметы, умный движок ценообразования, конвертация в счета |
| Финансы | Счета, оплаты Stripe, расходы, отчёты, маржа по заказам |
| Ресурсы | Мастера, материалы, транспорт, топливо |
| Клиенты | CRM, объекты, порталы самообслуживания |
| ИИ | Импорт нарядов, анализ PDF/email/фото, бизнес-помощник |
| Полевая работа | Мобильный интерфейс мастера, офлайн-синхронизация, GPS |
| SaaS | Мультитенантность, роли, подписки, RLS в Supabase |

### Технологический стек

- **Frontend:** React 19, TypeScript, Vite 8, Tailwind CSS 4
- **Состояние:** TanStack Query, React Hook Form, Zod
- **Backend:** Supabase (PostgreSQL, Auth, Storage, Edge Functions)
- **Платежи:** Stripe Checkout (счета и подписки)
- **Карты:** Google Maps API (диспетчерская, маршруты)
- **ИИ:** OpenAI через Edge Function proxy (или локальные fallback-ответы в demo)

## 2. Режимы работы

### Demo-режим (по умолчанию)

Активируется, если **не заданы** `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.

- Данные хранятся в `localStorage` браузера
- Вход с **любыми** email/паролем
- Демо-данные: клиенты, заказы, сметы, счета, материалы, автопарк
- Уведомления (email/SMS) эмулируются в очереди
- Оплата Stripe эмулируется локально
- Портал клиента доступен через «Демо-доступ к порталу» на странице входа

### Production-режим

При подключении Supabase:

- Данные в PostgreSQL с **Row Level Security (RLS)** по `company_id`
- Реальная аутентификация Supabase Auth
- Гибридный кэш: чтение дублируется в `localStorage` для офлайн-устойчивости
- Edge Functions: Stripe, OpenAI, email, SMS, webhooks
- Импорт демо-данных: **Настройки → Система → Импорт demo-данных**

Подробности: [DEPLOYMENT.md](./DEPLOYMENT.md)

## 3. Роли и права доступа

| Роль | Доступные модули | Стартовый маршрут |
|------|------------------|-------------------|
| **Owner** | Все модули (`*`) | `/dashboard` |
| **Admin** | Dashboard, заказы, наряды, сметы, CRM, расписание, диспетчерская, мастера, склад, транспорт, расходы, счета, отчёты, ИИ, настройки | `/dashboard` |
| **Dispatcher** | Dashboard, заказы, наряды, сметы, клиенты, объекты, расписание, диспетчерская, ИИ | `/dashboard` |
| **Technician** | Заказы (ограниченно), расписание, мобильный интерфейс `/tech` | `/tech` |
| **Accountant** | Dashboard, счета, расходы, отчёты, клиенты | `/dashboard` |
| **Customer** | Клиентский портал | `/portal/customer` |

**Онбординг владельца** (6 шагов) обязателен только для роли **Owner**. Приглашённые сотрудники (admin, dispatcher, technician, accountant) пропускают мастер настройки компании.

## 4. Быстрый старт

```bash
npm install
npm run dev          # http://localhost:5173
npm run verify:production
npm run test:e2e     # 134 E2E-теста
```

1. Откройте приложение и войдите с любыми данными (demo).
2. Пройдите **онбординг** (название компании, услуги, цены, сотрудник, транспорт, материалы).
3. Изучите **Панель** (`/dashboard`) — сводные KPI.
4. Создайте **заказ** (`/jobs`) или импортируйте **наряд** (`/work-orders`).
5. Для клиентского опыта: **Настройки → Клиенты → Скопировать ссылку на портал** или демо-портал с экрана входа.

## 5. Модули приложения

### 5.1. Панель руководителя (`/dashboard`)

**Назначение:** оперативный обзор бизнеса в реальном времени.

**Функционал:**
- KPI-карточки: выручка сегодня/за месяц, открытые и завершённые заказы, ожидающие сметы
- Затраты: труд, материалы, топливо, маржа прибыли
- Графики: динамика выручки и прибыли, структура расходов (pie), прибыльность услуг, эффективность мастеров
- Список последних заказов со статусами

**Возможности:**
- Сравнение с прошлым месяцем (тренды на карточках)
- Адаптивная вёрстка для планшетов и десктопа
- Данные агрегируются из заказов, расходов и топливных логов

---

### 5.2. Заказы (`/jobs`)

**Назначение:** центральный реестр всех работ — от черновика до завершения.

**Статусы заказа:** `draft` → `scheduled` → `in_progress` → `completed` | `on_hold` | `cancelled`

**Функционал:**
- Поиск по названию
- Фильтры-вкладки: все, запланированные, в работе, завершённые, черновики, **приостановленные**, **отменённые**
- Создание заказа: клиент, описание, приоритет, часы, выручка, даты
- Отображение: клиент, мастер, статус, приоритет, выручка, маржа, дата
- **Списание материалов** с склада на заказ (диалог у каждой строки)

**Массовые действия (bulk bar):**
| Действие | Описание |
|----------|----------|
| Смена статуса | Применить новый статус к выбранным |
| Назначить мастера | Bulk assign technician |
| Запланировать и назначить | Статус `scheduled` + мастер + дата |
| Отменить | Статус `cancelled` |
| Удалить | Безвозвратное удаление из хранилища |

**Выбор:** чекбоксы по строкам, «Выбрать все на странице».

---

### 5.3. Наряды и Vendor PO (`/work-orders`)

**Назначение:** импорт работ из внешних источников с помощью ИИ и обработка Vendor Purchase Orders.

#### Вкладка «ИИ-импорт»

| Режим | Входные данные | Результат |
|-------|----------------|-----------|
| PDF | Файл или вставленный текст | Извлечённые клиент, задачи, смета |
| Email | Текст письма | Структурированный наряд |
| Фото | Изображение повреждения | Тип работ, материалы, оценка труда |

После анализа: просмотр результатов, редактирование, создание сметы или заказа. История сохраняется в `work_orders`.

#### Вкладка «Vendor PO»

- Загрузка **одного или нескольких PDF** (batch)
- Парсинг полей: номер PO, приоритет, NTE, адрес, объём работ, compliance checklist
- Таблица записей с группировкой multi-site
- Экспорт в Excel
- **Создать заказ** / **Создать заказ + смету** из строки таблицы
- Демо-данные: кнопка загрузки 6 тестовых PO

---

### 5.4. Сметы (`/estimates`)

**Назначение:** коммерческие предложения с умным ценообразованием.

**Функционал:**
- **Умный движок (Smart Engine):** рекомендация цены на основе исторических заказов, каталога услуг, уверенности ИИ
- Варианты ценообразования: почасовая, фикс, срочность, выходные, скидка УК
- Создание сметы через форму: клиент, позиции (труд/материалы), наценка, срок действия
- Статусы: `draft`, `sent`, `approved`, `rejected`, `converted`
- **Отправить** клиенту (demo-уведомление в очередь)
- **Конвертировать в счёт** из отправленной/утверждённой сметы
- **Экспорт PDF** с локализацией RU/EN (позиции, итог, срок)

---

### 5.5. Клиенты (`/customers`)

**Назначение:** CRM — база заказчиков.

**Типы клиентов:** частный (`residential`), коммерческий (`commercial`), управляющая компания (`property_management`).

**Функционал:**
- Добавление/редактирование: имя, email, телефон, тип, адрес
- Поиск по таблице
- Метрики: количество заказов, суммарная выручка
- **Скопировать ссылку на портал** — magic link на 7 дней для клиентского портала

---

### 5.6. Объекты (`/properties`)

**Назначение:** адреса и объекты, привязанные к клиентам.

**Функционал:**
- Карточки объектов: название, адрес, тип (квартира, таунхаус, коммерция и др.)
- Примечания по доступу (коды, ключи)
- Создание через форму
- Используются в заказах, маршрутизации и портале УК

**Типы объектов:** apartment, townhouse, single_family, commercial, multi_family.

---

### 5.7. Расписание (`/scheduling`)

**Назначение:** календарь работ и планирование выездов.

**Функционал:**
- Виды: день / неделя / месяц
- Доступность мастеров (свободен/занят)
- Форма «Запланировать заказ»: выбор черновика/on_hold, мастер, дата, время, длительность
- Добавление события в календарь
- **Оптимизация маршрута:** панель с остановками, экономией миль/минут, ссылка «Открыть в Google Maps»
- Demo-уведомление клиенту при планировании

---

### 5.8. Диспетчерская (`/dispatch`)

**Назначение:** Kanban-доска для оперативного управления потоком заказов.

**Колонки:** Черновик → Запланирован → В работе → Завершён

**Функционал:**
- **Drag-and-drop** карточек между колонками (смена статуса)
- Смена статуса через выпадающий список на карточке
- При переводе в `scheduled`: demo SMS мастеру и email клиенту
- Аудит-лог действий диспетчера
- **Карта объектов:** Google Maps или fallback со ссылками на адреса
- **Панель маршрута:** оптимизированные остановки, открытие маршрута в Maps

---

### 5.9. Мастера (`/technicians`)

**Назначение:** учёт стоимости сотрудников и расчёт реальной почасовой ставки.

**Функционал:**
- Карточки команды: зарплата, налоги, страховка, льготы, накладные
- **Реальная стоимость** (true cost) и **ставка клиенту** (billing rate)
- Маржа по сотруднику
- Добавление мастера через форму
- Данные используются в назначении на заказы и отчётах по эффективности

---

### 5.10. Материалы (`/materials`)

**Назначение:** складской учёт и ценообразование материалов.

**Функционал:**
- Таблица: название, категория, поставщик, себестоимость, цена продажи, остаток, reorder level
- **Предупреждение о низком запасе** (ниже минимального уровня)
- **Приёмка на склад** (receive stock) — увеличение количества
- Создание материала через форму
- **Списание на заказ** — из модуля «Заказы» (Job Material Usage)
- Транзакции инвентаря в журнале

---

### 5.11. Транспорт и топливо (`/vehicles`)

**Назначение:** управление автопарком и учёт топлива.

**Функционал:**
- Карточки ТС: название, тип, марка/модель, госномер, пробег
- Журнал заправок: дата, ТС, мили, галлоны, цена, сумма
- KPI: расходы на топливо за месяц, стоимость за милю
- Создание ТС через форму
- Топливные расходы учитываются в отчётах и структуре затрат

---

### 5.12. Расходы (`/expenses`)

**Назначение:** операционные расходы бизнеса.

**Функционал:**
- Категории: Fuel, Materials, Tools, Insurance, Office, Other
- Поля: дата, описание, сумма, привязка к заказу/ТС (опционально)
- Итог за текущий месяц
- Добавление через форму
- Участвуют в pie-chart «Структура расходов» на панели и в отчётах

---

### 5.13. Счета (`/invoices`)

**Назначение:** выставление счетов и приём оплат.

**Функционал:**
- Сводка: неоплаченные, оплачено за месяц, всего счетов
- Статусы: `draft`, `sent`, `paid`, `overdue`, `partial`
- Создание счёта, позиции, налог, срок оплаты
- **Отправить** клиенту (demo email)
- **Demo-оплата** / **Stripe Checkout** (при настройке)
- **Экспорт PDF** RU/EN: номер, позиции, подытог, налог, баланс
- Конвертация из утверждённой сметы

---

### 5.14. Отчёты (`/reports`)

**Назначение:** финансовая и операционная аналитика с экспортом.

**Фильтр периода:** дата «С» / «По» (по умолчанию последние 6 месяцев).

**Вкладки:**

| Вкладка | Содержание |
|---------|------------|
| **Финансы** | График выручки и прибыли по месяцам |
| **Прибыльность заказов** | Список завершённых заказов: выручка, затраты, маржа |
| **Мастера** | Эффективность: выручка, заказы, % |
| **Клиенты** | Выручка и число заказов по клиенту |
| **Услуги** | Прибыльность по категориям услуг (ключевые слова в названии) |
| **Расходы** | Pie-chart структуры затрат |

**Сводка в шапке:** число заказов, выручка, чистая прибыль, маржа %, сравнение с предыдущим периодом.

**Экспорт:**
- **PDF** — локализованный отчёт по активной вкладке (RU/EN)
- **CSV/Excel** — полный отчёт (заказы, мастера, услуги, клиенты)

---

### 5.15. ИИ-бизнес-помощник (`/ai-assistant`)

**Назначение:** диалоговый анализ бизнес-данных на естественном языке.

**Функционал:**
- Чат-интерфейс с приветствием на текущем языке (RU/EN)
- Готовые вопросы (suggested prompts): прибыльность, мастера, заказы
- Произвольные вопросы в поле ввода
- В demo: fallback-ответы из `ai-fallbacks.ts` без OpenAI
- В production: запросы через Edge Function `openai-proxy` (ключ не в браузере)

**Примеры запросов:** «Какой мастер самый прибыльный?», «Покажи убыточные заказы», «Рекомендации по ценообразованию».

## 6. Порталы для клиентов

Язык интерфейса следует `handymanos_locale` (RU/EN).

**Для:** property management — подача заявок на работы.

- Просмотр активных заявок

- Заявка создаёт заказ в системе (demo flow)

## 8. Онбординг и приглашения

### 8.1. Онбординг владельца (`/onboarding`) — 6 шагов

1. **Компания** — название, контакты, адрес  
2. **Услуги** — выбор из каталога (гипсокартон, сантехника, электрика…)  
3. **Цены** — почасовая ставка, множители срочности/выходных, скидка УК  
4. **Сотрудники** — первый мастер (имя, роль, зарплата, ставка)  
5. **Транспорт** — первое ТС  
6. **Материалы** — предзагруженный список (можно пропустить)

После завершения → `/dashboard`. Прогресс сохраняется в `localStorage`.

### 8.2. Онбординг мастера (`/tech-onboarding`)

Краткая форма для приглашённого technician: имя, телефон, навыки → `/tech`.

### 8.3. Приглашения в команду

- Owner/Admin создаёт invite в **Настройки → Команда**
- Ссылка `/login?invite=TOKEN`
- Новый пользователь регистрируется; существующий — входит и присоединяется к компании
- Роли: admin, dispatcher, technician, accountant

## 9. Общие возможности интерфейса

### 9.1. Локализация (i18n)

- Языки: **русский** и **английский**
- Переключатель в сайдбаре и в настройках
- Локализованы: навигация, формы, PDF-экспорты, порталы, error boundary
- Ключ хранения: `handymanos_locale`

### 9.2. Command Palette (`Ctrl+K` / `Cmd+K`)

- Быстрая навигация по разделам
- Глобальный поиск: заказы, клиенты, счета
- Переключение темы

### 9.3. Глобальный поиск (header)

Поиск сущностей с переходом на соответствующую страницу.

### 9.4. Мультикомпания (Company Switcher)

- Переключение между компаниями, доступными пользователю (`company_members`)
- Изоляция данных по `company_id`
- Demo: две тестовые компании

### 9.5. PWA и офлайн

- `manifest.json` + Service Worker (`/sw.js`)
- Офлайн-очередь для мобильного мастера
- Установка на домашний экран (mobile)

### 9.6. Тема оформления

Светлая / тёмная тема (`class="dark"` на `<html>`), сохранение в `localStorage`.

### 9.7. Обработка ошибок

- React Error Boundary с локализованным UI
- Кнопка «Перезагрузить»
- Отправка отчёта в Sentry / webhook (при настройке)

## 12. Безопасность и аудит

- **RLS** на всех таблицах PostgreSQL — доступ только к данным своей компании
- **Роли** ограничивают видимость модулей в UI
- **Portal tokens** с TTL для внешних пользователей
- **Rate limiting** на Edge Functions (`check_rate_limit`)
- **Audit log** — действия пользователей (диспетчеризация, изменения)
- **Приватный Storage bucket** `handymanos` для фото заказов
- RPC: `get_portal_estimates`, `portal_submit_review`, `get_accessible_companies`
